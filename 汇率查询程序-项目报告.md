# 汇率查询与推送程序项目报告

## 项目概述

本项目是一个纯 Python 3 命令行工具，从韩国 SMBS 和中国外汇交易中心获取汇率，支持表格输出、JSON 输出以及向外部 API 推送。

- 版本：1.1
- 更新日期：2026-07-19
- 运行环境：Python 3.6+
- 第三方运行依赖：无

## 数据流程

```text
命令行 / config.json
          |
          v
     确定查询日期
          |
          v
  SMBS 获取与解析 ---- 无有效汇率 --> 停止
          |
          v
Chinamoney 获取与解析 -- 非交易日 --> 跳过
          |
          v
   组装带日期的数据
          |
          +--> 表格
          +--> JSON
          +--> HTTP POST
```

## 数据源

### SMBS

- 内容：外币兑韩元牌价。
- 格式：EUC-KR 编码的 query string。
- 转换：`CNH` 统一输出为 `CNY`。
- 完整性：解析后必须至少存在一条汇率。

### Chinamoney

- 内容：人民币汇率中间价。
- 格式：UTF-8 JSON。
- `100JPY/CNY` 类货币对按 100 单位换算。
- `CNY/MOP` 类货币对取倒数，统一输出为外币兑 CNY。
- 不包含 CNY 的货币对不输出。

## 数据协议

表格、JSON 和推送模式共用同一数据结构：

```json
{
  "date": "2026-07-18",
  "smbs": [
    {"from_currency": "USD", "to_currency": "KRW", "rate": 1510.0}
  ],
  "chinamoney": [
    {"from_currency": "USD", "to_currency": "CNY", "rate": 7.2}
  ]
}
```

- `date` 为查询日期，格式 `YYYY-MM-DD`。
- `rate` 为 JSON number，最多保留 8 位小数。
- Chinamoney 无可用数据时省略 `chinamoney`。

## 配置与推送

- 命令行 `--date` 优先于 `config.json` 中的 `date`。
- 推送仅支持 `POST application/json`。
- `timeout` 必须为正数。
- 真实 `config.json` 只保留在本机，Git 仅跟踪 `config.example.json`。
- 配置和日志相对于程序所在目录定位；PyInstaller EXE 使用 EXE 所在目录。

## 错误处理

- 每个网络请求最多尝试 5 次，重试间隔 60 秒。
- SMBS 获取失败或解析为空时退出。
- Chinamoney 失败时跳过，不影响 SMBS 输出。
- 获取和推送失败信息写入 `logs/`。

## 测试

项目使用 Python 标准库 `unittest`，离线覆盖：

- SMBS 解析与空数据。
- CNH 币种映射。
- Chinamoney 单位换算与货币对翻转。
- 非 CNY 货币对过滤。
- 日期优先级与格式校验。
- 推送方法校验。
- SMBS 空数据短路。

```bash
python3 -m unittest discover -s tests -v
```

## 已知边界

- SMBS 端点当前使用 HTTP，需由数据源确认是否支持 HTTPS。
- 两个数据源仍按顺序请求，极端失败情况下等待时间较长。
- 实际 API 推送需要目标服务可达，离线测试不会发送网络请求。
